   286→   286→# 4. LOAD SCREEN ANIMATION
   287→   287→
   288→   288→## 4.1 Purpose
   289→   289→
   290→   290→The load screen serves two functions:
   291→   291→1. **Brand moment** — Logo animation establishes premium feel
   292→   292→2. **Frame preload** — Hero canvas requires pre-rendered frames
   293→   293→
   294→   294→## 4.2 Sequence Overview
   295→   295→
   296→   296→```
   297→   297→Phase 1: Logo Pulse     →  Logo scales 1.0 ↔ 1.05, subtle glow pulse, small size (60px)
   298→   298→Phase 2: Bounce Reveal  →  Whole webpage bounces up from bottom, centered in view
   299→   299→Phase 3: Experience     →  Hero section begins immediately
   300→   300→```
   301→   301→
   302→   302→## 4.3 Visual Design
   303→   303→
   304→   304→```
   305→   305→┌──────────────────────────────────────────────────────────────┐
   306→   306→│                                                              │
   307→   307→│                                                              │
   308→   308→│                                                              │
   309→   309→│                    [ODL Logo]                                 │
   310→   310→│                    (centered, 60px wide)                      │
   311→   311→│                    (subtle pulse animation)                   │
   312→   312→│                                                              │
   313→   313→│                                                              │
   314→   314→│                                                              │
   315→   315→│                                                              │
   316→   316→└──────────────────────────────────────────────────────────────┘
   317→   317→```
   318→   318→
   319→   319→- **Background:** `#FFFFFF` (white)
   320→   320→- **Logo:** Centered, 60px wide, subtle pulse animation
   321→   321→- **No progress bar**
   322→   322→- **No percentage text**
   323→   323→
   324→   324→## 4.4 Transition
   325→   325→
   326→   326→- **Load → Content:** Logo fades out (0.3s), then entire webpage bounces up from bottom (`translateY(100vh)` to `translateY(0)`) with spring animation (`stiffness: 200, damping: 20`)
   327→   327→- The bounce effect brings the full page into view while continuing the experience
   328→   328→- Hero section activates immediately after reveal
   329→   329→
   330→   330→## 4.5 Technical Implementation
   331→   331→
   332→   332→```typescript
   333→   333→// Bounce-in animation for page reveal
   334→   334→const pageVariants = {
   335→   335→  hidden: { y: "100vh", opacity: 0 },
   336→   336→  visible: {
   337→   337→    y: 0,
   338→   338→    opacity: 1,
   339→   339→    transition: {
   340→   340→      type: "spring",
   341→   341→      stiffness: 200,
   342→   342→      damping: 20,
   343→   343→      mass: 1,
   344→   344→    },
   345→   345→  },
   346→   346→};
   347→   347→```
   348→   348→
   349→   349→## 4.6 Fallback
   350→   350→
   351→   351→- If user has `prefers-reduced-motion: reduce`, skip bounce, instant reveal
   352→   352→- Maximum load screen duration: 5 seconds (force dismiss)
   353→   353→- Loading state stored in sessionStorage to avoid replay on SPA navigation

   404→   404→### Animation System
   405→   405→
   406→   406→This section uses a **scroll-driven card morph** — NOT a frame-sequence canvas.
   407→   407→
   408→   408→**How it works:**
   409→   409→1. 20 flip cards start scattered across the viewport
   410→   410→2. On load, they animate into a circle formation
   411→   411→3. On scroll, the circle morphs into a bottom arc
   412→   412→4. Cards shuffle along the arc as user continues scrolling
   413→   413→5. Mouse parallax affects card positions
   414→   414→6. Each card flips on hover to reveal ODL service labels
   415→   415→
   416→   416→**Scroll math:**
   417→   417→- Morph progress: `scrollY [0, 600]` → `[0, 1]` (circle to arc)
   418→   418→- Arc rotation: `scrollY [600, 3000]` → `[0, 360]` (shuffle cards)
   419→   419→- Content fade: `morphProgress [0.8, 1]` → `[0, 1]`
   420→   420→- CTA appearance: `morphProgress > 0.9`
   421→   421→

   976→   976→## 11.1 Architecture
   977→   977→

   983→   983→## 11.2 Unique Animation Per Section
   984→   984→
   985→   985→**Every section has a distinct animation — no repeating patterns:**
   986→   986→
   987→   987→| Section | Animation |
   988→   988→|---------|-----------|
   989→   989→| Hero | Card morph (scatter → circle → arc) + mouse parallax |

   991→   991→| Trust Bar | Liquid glass overlay + number counters |
   992→   992→| Services | 3D card tilt + frosted glass hover |
   993→   993→| How It Works | Vertical zigzag timeline with progressive reveal |
   994→   994→| Problem/Solution | Overlay revelation (solution slides over problem) |
   995→   995→| Testimonials | Stacking cards with scroll-driven pile |

   999→   999→## 11.3 Easing Tokens
  1000→  1000→
  1001→  1001→```css
  1002→  1002→:root {
  1003→  1003→  --ease-premium: cubic-bezier(0.32, 0.72, 0, 1);
  1004→  1004→  --ease-spring: cubic-bezier(0.16, 1, 0.3, 1);
  1005→  1005→  --ease-sharp: cubic-bezier(0.5, 0, 0.75, 0);
  1006→  1006→}
  1007→  1007→```
  1008→  1008→
  1009→  1009→## 11.4 Performance Rules
  1010→  1010→
  1011→  1011→- Animate only `transform` and `opacity`
  1012→  1012→- `will-change: transform` only on actively animating elements
  1013→  1013→- Canvas: DPR-aware sizing, passive listeners, RAF-guarded updates
  1014→  1014→- `prefers-reduced-motion: reduce` → disable all animations
